🚨 EXAM EXIT DETECTION (Anti-Cheat + Auto-Submit):
- On the exam page (exam.html), detect if the student leaves 
  the exam in any of these ways:
    1. Closes the browser tab or window (beforeunload event)
    2. Navigates away (visibilitychange event — tab switch)
    3. Presses browser Back button
    4. Refreshes the page

- On first violation (tab switch / visibility change):
    - Show a full-screen red warning overlay:
        ⚠️ "Warning! Do not leave the exam window.
            This activity has been recorded."
    - Resume exam when they return, but log the violation

- On exit/close/refresh (beforeunload) OR second violation:
    - Immediately auto-submit whatever answers have been filled
    - Calculate score for answered questions (unanswered = wrong)
    - Save result to localStorage 'results' array as normal
    - Add a special notification to 'notifications' array for teacher:
      {
        studentName, studentClass, examTitle,
        score, total, percentage, timestamp,
        exitViolation: true,
        violationType: "Tab switched" | "Browser closed" | "Page refreshed",
        read: false
      }
    - Redirect student to results.html with a message:
        "Exam auto-submitted due to exit. Your result has been recorded."

- In the Teacher Notification Panel:
    - Notifications where exitViolation: true should show a 
      distinct RED WARNING badge labeled "⚠️ Exam Exited"
    - Show the violation type below the student name
    - Sort these to appear at the TOP of the notification list
    - The badge count should still increment for these

- On the Results Page for the student:
    - If the result has exitViolation: true, show a subtle 
      red banner at the top:
        "⚠️ This exam was auto-submitted because you exited 
         the exam window.
